# tg-form-filler

Telegram-бот, который принимает сообщение от администратора, извлекает данные с помощью LLM (DeepSeek) и заполняет Google Формы. Встроенный AI-нутрициолог анализирует приёмы пищи и даёт обратную связь.

## Возможности

- Автоматически определяет нужную форму по тексту сообщения
- Поддерживает несколько форм одновременно (дневник питания, дневник расходов)
- Категория приёма пищи определяется автоматически по текущему времени (МСК), если не указана явно
- Напоминает о записи приёма пищи, если прошло более 8 часов — только в дневные часы (08:00–22:00 МСК)
- Ведёт статистику расходов в памяти и каждое утро в 9:00 МСК отправляет краткий LLM-отчёт о тратах за прошлый день
- AI-нутрициолог комментирует каждый приём пищи и формирует периодические отчёты о питании
- Распознаёт записи-напитки (поле «Напиток») и не комментирует их, а отчёты их игнорируют
- Если новая запись приходит в течение 10 минут после предыдущей — предлагает объединить её с прошлой (порции суммируются, описание дополняется)

## Нутрициолог

После каждой записи в дневник питания бот подгружает последние 10 приёмов пищи из Google Sheets и отправляет короткий комментарий (1–2 предложения) от AI-нутрициолога. Анализ учитывает контекст предыдущих приёмов за день и не повторяет уже данные советы (в памяти хранятся последние 10 комментариев).

Оценка строится на наборе правил из `rules/nutrition_rules.md`:

- соотношение растительной и животной пищи (цель ≥ 70% растительной)
- контроль порций и ультраобработанных продуктов
- качество белков и жиров, потребление сахара
- разнообразие и цветовая палитра овощей/фруктов
- наличие ферментированных продуктов и клетчатки
- режим питания и ночное окно голодания (≥ 12 ч)

Тон — дружеский, но прямой: хорошие приёмы хвалит с энтузиазмом, плохие критикует конкретно и предлагает альтернативы.

Помимо мгновенной обратной связи бот автоматически формирует периодические отчёты:

| Отчёт       | Расписание                     | Содержание                                                        |
|--------------|--------------------------------|-------------------------------------------------------------------|
| Дневной      | каждый день в 00:00 МСК       | что было хорошо, что улучшить, 2–3 рекомендации на завтра         |
| Недельный    | понедельник в 00:05 МСК       | тренды, повторяющиеся проблемы, прогресс, 3–5 рекомендаций        |
| Месячный     | 1-е число месяца в 00:10 МСК  | долгосрочные паттерны, достижения, стратегические рекомендации     |

## Настройка

Скопируй `.env.example` → `.env` и заполни переменные:

```
DEEPSEEK_API_KEY=
TG_BOT_TOKEN=
TG_ADMIN_CHAT_ID=
FOOD_DIARY_SPREADSHEET_ID=
```

Скопируй `.example` конфиги из директории `form_configs` и заполни их реальными `form_url` и `entry.*` идентификаторами полей.

### Поле «Напиток»

В примере конфигурации `food_diary_form_config.example.json` есть поле `Напиток` (Да/Нет). Чтобы оно заработало:

1. Добавь в Google Форму радио-поле «Напиток» с вариантами «Да» и «Нет».
2. Добавь столбец «Напиток» в Google Таблицу с приёмами пищи.
3. Допиши соответствующее поле с реальным `entry_id` в свой `food_diary_form_config.json`.
4. На странице статистики Google Таблицы исключай строки с `Напиток = Да`, чтобы напитки не считались приёмами пищи.

Когда поле настроено, бот:
- помечает запись как напиток через LLM-определение,
- пропускает комментарий нутрициолога для напитков,
- исключает напитки из дневного / недельного / месячного отчётов.

## Локальный запуск

```bash
docker compose up -d
```

## Деплой на сервер

На сервере нужен только Docker.

Впиши необходимые алиасы в `deploy/deploy.sh` и запусти:

```bash
./deploy/deploy.sh
```
